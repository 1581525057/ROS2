## 问题：为什么不能在回调里直接做耗时操作？

ROS2 的回调函数（订阅回调、定时器回调）都运行在同一个事件循环里。
如果在回调里做耗时操作（比如语音朗读），事件循环会被阻塞，
新的消息就无法及时处理。

---

## 解决方案：生产者-消费者模式

```
[ROS 回调] ──put──► [Queue 队列] ──get──► [后台线程]
（快速返回）         （缓冲区）            （慢慢处理）
```

代码骨架：

```python
from queue import Queue
from threading import Thread

class MyNode(Node):
    def __init__(self, node_name):
        super().__init__(node_name)
        self.work_queue = Queue()

        # 订阅话题
        self.create_subscription(String, 'topic', self.callback, 10)

        # 启动后台处理线程
        t = Thread(target=self.worker)
        t.daemon = True   # 主程序退出时线程自动结束
        t.start()

    def callback(self, msg):
        # 只放队列，立刻返回
        self.work_queue.put(msg.data)

    def worker(self):
        while rclpy.ok():
            if self.work_queue.qsize() > 0:
                item = self.work_queue.get()
                # 在这里做耗时操作（朗读、IO、计算等）
                do_something_slow(item)
            else:
                time.sleep(0.1)   # 避免空转
```

## Python 多线程下载练习（CHAPT2）

```python
# learn_thread.py
class Download:
    def download(self, url, callback):
        response = requests.get(url, timeout=5)
        response.encoding = 'utf-8'
        callback(url, response.text)

    def start_download(self, url, callback):
        t = threading.Thread(target=self.download, args=(url, callback))
        t.start()

def word_count(url, result):
    print(f"下载完成：{url}，长度：{len(result)}")

download = Download()
download.start_download("http://127.0.0.1:8000/novel1.txt", word_count)
download.start_download("http://127.0.0.1:8000/novel2.txt", word_count)
```

三个 `start_download` 同时运行，互不等待，体现并发下载。
